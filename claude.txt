## Critical Rules for Claude

1. **String literals use double quotes only:**
- ✅ `[literal:string:"hello"]`
- ❌ `[literal:string:hello]` — Blockify now recovers bare words, but always use canonical form
- ❌ `[literal:string:'hello']` — single quotes are not recovered

2. **Numbers are unquoted:**
- ✅ `[literal:number:42]` and `[literal:number:-108]`
- ❌ `[literal:number:"42"]`

3. **All opcodes must match exactly** — typos break rendering:
- ✅ `looks_say`
- ❌ `look_say` (missing 's')

4. **Arithmetic operators use NUM1/NUM2, not OPERAND1/OPERAND2:**
- ✅ `operator_add inputs:{NUM1:..., NUM2:...}`
- ❌ `operator_add inputs:{OPERAND1:..., OPERAND2:...}`

5. **All IDs within a root must be unique**

6. **Validate constraints before returning:**
- No circular references
- No stack nodes in `inputs` — stacks only go in `stacks:{}`
- No opcode nodes in `stacks` — only stack nodes
- No fields outside `fields:{}`

## Common Failure Modes (Based on Testing)

- **String literal quoting** — Most common issue. Models produce unquoted strings. Blockify now recovers bare words silently, but canonical form (double-quoted) is always required.
- **Opcode misspelling** — Abbreviations like `look_say` instead of `looks_say`.
- **Arithmetic operator confusion** — Using `OPERAND1`/`OPERAND2` for arithmetic instead of `NUM1`/`NUM2`.
- **Session context bleed** — When running multiple mutations, re-anchor by explicitly stating "start from here, don't change anything: (IR)"
- **Unintended structural changes** — Preserving unrelated structure is required by AI Mutation Rules.

## Task Types

- **Mutation:** "Take this IR and change X to Y" — parse, find target, apply change, validate output
- **Generation:** "Create IR for blocks that do X" — synthesize IR, validate against grammar
- **Analysis:** "What does this IR do?" — walk tree, describe each block
- **Transformation:** "Refactor this IR by X" — restructure while preserving semantics
